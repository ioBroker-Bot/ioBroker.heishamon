# Protokoll-Analyse

Dieses Verzeichnis wird in **Phase 0** ([../plan/phase-0-recon.md](../plan/phase-0-recon.md)) gefüllt. Es ist die kanonische Referenz für das CN-CNT-Protokoll der Panasonic Aquarea, abgeleitet aus dem HeishaMon-Code und gegen die Doku des HeishaMon-Repos abgeglichen.

## Geplante Inhalte (entstehen in Phase 0)

- `code-map.md` — welche Dateien im HeishaMon-Repo sind relevant, welche Funktionen erledigen was
- `frames.md` — Frame-Layouts (Haupt-Poll, Set-Commands, Optional-PCB-Antworten)
- `crc.md` — Checksum-Algorithmus, verifiziert
- `datapoints.md` — vollständige Datenpunktliste mit Offsets, Encodings, Wertebereichen
- `doc-vs-code-diff.md` — wo Doku und Code des HeishaMon-Repos auseinanderlaufen
